Welcome to SMART

This guided demo will walk you through the steps involved in Defining, Deploying & Using a Product Listing. It will also show you how to add & list the products.

This product listing is created as a simple SMART FLOW defined with one Prime Object, which is the definition of the product. You will be able to Store, Retrieve & Search the products with Standard SMART events.

In general you can define any data to be stored as a prime object and use the standard SMART events to store/retrieve. This is applicable for usecases that involve listing/searching such as list of services, doctors, dealers, products, schools and many more.

In the next few steps we will create the product Flow and Deploy it in SMART+.

Flow Details

Name of your SMART Flow

SMART flows are enabled for a Tenant associated with an ID. This ID uniquely identifies your run time environment and can be accessed only by you. The tenant ID has been auto generated by the wizard based on your Email id. SMART flows are identified using a unique identifier "Flow Name".

In this demo we will name the flow as ProductDemo1234(Random number)



Prime Object Details

Name your Prime Object

A SMART flow is associated with a Prime Object, which is typically the data to be listed. The Prime Object is identified using a name.

In this demo our prime object is named "Product".

Prime Object Definition

Define your Prime Object attributes

Prime objects in a SMART flow is defined by a set of attributes. An attribute takes a label and its type. One of the attributes has to be marked as a key. When data is added as for this prime object, the key will uniquely identify the data. Pre-defined templates are provided for commonly used Prime Object definitions. This can be customized by adding/deleting the attributes.

In this demo we will use a pre-defined product listing template "Basic Template".
